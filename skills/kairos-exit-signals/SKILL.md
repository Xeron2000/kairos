# 出场信号 - Bit浪浪出场信号

## 核心概念

出场信号：
- **全包回来**：大阴线吞没涨幅
- **该突破不突破**：完美结构未能兑现
- **丧失龙头地位**：资金转移到新龙头
- **大盘见顶**：BTC到达波段顶部

## 使用场景

当需要判断是否平仓时使用此skill：
- 识别出场信号
- 评估持仓风险
- 决定平仓时机

## CLI命令

```
MCP tool: check_exit_signals
参数: symbol (string)

返回示例:
  symbol: BTC/USDT
  position: LONG
  entry_price: 65000
  current_price: 68500
  pnl_pct: 5.4
  signals: ["全包回来"]
  recommendation: 考虑减仓
  action: 减仓50%
```

## 出场信号类型

### 1. 全包回来
**特征：**
- 价格向上强势拉升突破后
- 没有在高位稳住
- 一根大阴线将涨幅全部跌回

**含义：**
- 多头动能衰竭
- 进入深度调整
- 应该立即离场

**应对：**
- 立即平仓或大幅减仓
- 不能死扛
- 等待新信号

### 2. 该突破不突破
**特征：**
- 价格经历完美、充分的横盘收敛
- 来到多头理应爆发的关键节点
- 打出假突破并拐头向下

**含义：**
- 主力做多意愿不足
- 后续大概率陷入调整甚至深跌
- 必须离场

**应对：**
- 立即平仓
- 不要抱有幻想
- 等待新结构

### 3. 丧失龙头地位
**特征：**
- 资金永远喜新厌旧
- 市场出现另一个涨幅更猛、结构更好、更吸金的新龙头
- 原来的币种丧失龙头地位

**含义：**
- 资金转移到新龙头
- 原龙头可能滞涨或下跌
- 应该跟随最强合力

**应对：**
- 果断出局
- 跟随新龙头
- 不要留恋旧龙头

### 4. 大盘见顶
**特征：**
- BTC向上没有空间
- 触及强压力位
- 出现"见顶撤退"信号

**含义：**
- 山寨币行情高度依赖大盘
- 大饼一旦回调，绝大多数山寨币都会被带崩
- 哪怕形态再好也要跑路

**应对：**
- 果断跑路
- 不要心存侥幸
- 等待大盘企稳

## 出场策略

### 分批出场
- 不追求卖在最高点
- 关键压力位分批平仓
- 例如目标30和35，可以在29开始出货

### 信号出场
- 出现出场信号立即行动
- 不要犹豫
- 保本第一

### 止损出场
- 止损触发立即执行
- 不要移动止损
- 严格纪律

## LLM判断流程

hermes agent使用此skill时：
1. 调用 MCP tool `check_exit_signals` 获取出场分析
2. 检查是否出现出场信号
3. 评估持仓风险
4. 决定平仓时机和仓位
5. 执行出场操作

## 注意事项

- 出场信号比进场信号更重要
- 保本第一，盈利第二
- 不要心存侥幸
- 严格纪律

## 实战案例

### 全包回来案例：SOL/USDT
- **持仓**：150做多，盈利10%
- **信号**：大阴线从165跌到150（全包回来）
- **行动**：立即平仓
- **结果**：保住了5%盈利，避免了后续20%下跌

### 该突破不突破案例：AVAX/USDT
- **持仓**：35做多，盈利8%
- **信号**：完美箱体收敛，但假突破后下跌
- **行动**：立即平仓
- **结果**：保住了3%盈利，避免了后续15%下跌

### 丧失龙头地位案例：LINK/USDT
- **持仓**：15做多，盈利12%
- **信号**：SOL成为新龙头，资金转移
- **行动**：平仓LINK，转投SOL
- **结果**：LINK滞涨，SOL上涨30%

### 大盘见顶案例：所有山寨币
- **持仓**：多个山寨币多头
- **信号**：BTC触及70,000强压力位
- **行动**：全部平仓
- **结果**：BTC回调15%，山寨币普遍下跌30%+
