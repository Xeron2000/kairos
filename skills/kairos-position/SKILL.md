# 仓位管理 - kairos仓位管理原则

## 核心概念

kairos仓位管理：
- **固定仓位**：总资金等份划分，每次只用固定金额
- **低倍杠杆**：BTC≤10倍，山寨≤5倍
- **严格分仓**：亏损从场外补，盈利提现

## 使用场景

当需要管理仓位时使用此skill：
- 计算仓位大小
- 查看当前持仓
- 管理风险

## CLI命令

```bash
# 查看仓位状态
pwatch position status

# 计算仓位大小
pwatch position size --capital 10000 --risk-pct 33 --leverage 5

# 查看交易历史
pwatch position history --limit 20

# 查看策略统计
pwatch position stats --strategy box_breakout
```

## 仓位计算

### 固定仓位法
- 总资金等份划分（如3份）
- 每次只用1份开仓
- 亏损从场外补，盈利提现

### 杠杆限制
- **BTC/ETH**：最多10倍
- **山寨币**：最多5倍
- **套利**：最多3倍

### 仓位计算公式
```
仓位大小 = (资金 × 风险比例) / (止损距离 / 进场价)
```

## 风险约束

### 单笔风险
- 最大单笔仓位：33%
- 最大单笔亏损：3%（止损10%距离）

### 总体风险
- 最大总暴露：66%（2个仓位）
- 最大同时持仓：2个
- 单日连续亏损：3次暂停

## 加仓策略

### 二次加仓条件
- 必须已有成本极佳的底仓
- 底仓有利润作为缓冲
- 趋势明确，主升浪进行中
- 结构完美，二次突破或企稳

### 加仓风险
- 高位突破加仓极易被洗盘
- 严格控制加仓仓位
- 流动性差的币种不能重仓

## 止损设置

### 结构止损
- 箱体突破：止损放在箱体下沿
- 双底拐点：止损放在双底最低点
- 追涨单：止损放在最近有效支撑

### 推原价保护
- 行情脱离成本区后
- 迅速将止损位上调至开仓均价
- 确保该笔交易不再亏损

## 止盈策略

### 分批出货
- 不追求卖在最高点
- 关键压力位分批平仓
- 例如目标30和35，可以在29开始出货

### 出场信号
- **全包回来**：大阴线吞没涨幅
- **该突破不突破**：完美结构未能兑现
- **丧失龙头地位**：资金转移到新龙头
- **大盘见顶**：BTC到达波段顶部

## LLM管理流程

hermes agent使用此skill时：
1. 调用 `pwatch position status` 获取当前持仓
2. 根据风险约束计算仓位大小
3. 设置止损和止盈目标
4. 执行交易并记录

## 注意事项

- 固定仓位是心态的托底
- 低倍杠杆是生存的关键
- 止损是保命的底线
- 分批止盈是利润的保障
