# 加仓策略 - Bit浪浪二次加仓方法

## 核心概念

二次加仓策略：
- **底仓优势是前提**：必须已有成本极佳的底仓
- **寻找企稳拐点**：不能在价格直线上涨时盲目追高
- **严格控制仓位**：加仓仓位不能过重

## 使用场景

当需要加仓时使用此skill：
- 判断是否满足加仓条件
- 寻找加仓时机
- 控制加仓风险

## CLI命令

```
MCP tool: check_pyramiding

返回示例:

  signal: 二次突破
  suggested_entry: 69000
  stop_loss: 67500
```

## 加仓条件

### 必要条件
1. **已有底仓**：必须持有成本极佳的底仓
2. **底仓有利润**：底仓必须处于盈利状态
3. **趋势明确**：主升浪正在进行
4. **结构完美**：二次突破或企稳结构

### 充分条件
1. **二次突破**：价格突破前高
2. **企稳拐点**：回调后不创新低
3. **收敛突破**：箱体收敛后突破
4. **放量确认**：成交量放大确认

## 加仓方法

### 二次突破加仓
**条件：**
- 价格突破前高
- 成交量放大
- 趋势明确

**进场：**
- 突破确认后进场
- 等待回踩确认

**止损：**
- 前低或突破起点

### 企稳拐点加仓
**条件：**
- 价格回调不创新低
- 形成双底结构
- 向上拐头

**进场：**
- 拐点确认后进场
- 等待反弹确认

**止损：**
- 双底最低点

### 收敛突破加仓
**条件：**
- 价格在箱体内收敛
- 收敛程度高
- 向上突破

**进场：**
- 突破确认后进场
- 等待回踩确认

**止损：**
- 箱体下沿

## 风险控制

### 加仓风险
1. **高位洗盘**：高位突破加仓极易被洗盘
2. **仓位过重**：加仓后总仓位过重
3. **流动性风险**：深度差的币种难平仓
4. **心态失衡**：加仓亏损导致心态失衡

### 风险控制措施
1. **轻仓加仓**：加仓仓位不超过原仓位30%
2. **严格止损**：加仓后整体止损必须设置
3. **流动性检查**：确保币种有足够流动性
4. **心态管理**：保持平和心态

## 加仓后管理

### 止损调整
- 加仓后整体止损必须设置
- 可以将止损上移至成本价
- 确保整体仓位不亏损

### 止盈策略
- 分批止盈，不追求最高点
- 关键压力位分批平仓
- 保持部分仓位跟踪趋势

## LLM判断流程

hermes agent使用此skill时：
1. 调用 MCP tool `check_pyramiding` 获取加仓分析
2. 检查是否满足加仓条件
3. 寻找加仓时机
4. 计算加仓仓位
5. 设置止损和止盈

## 注意事项

- 加仓风险较高，必须谨慎
- 底仓优势是前提
- 严格控制仓位
- 严格止损是关键

## 实战案例

### 成功加仓案例：BTC/USDT
- **底仓**：65,000做多，仓位10,000 USDT
- **当前价格**：68,500（+5.4%）
- **加仓信号**：突破69,000
- **加仓仓位**：3,000 USDT（30%）
- **加仓价格**：69,000
- **整体成本**：65,800
- **整体止损**：64,000
- **目标**：75,000
- **结果**：成功到达目标，盈利14.0%

### 失败加仓案例：ETH/USDT
- **底仓**：3,200做多，仓位10,000 USDT
- **当前价格**：3,400（+6.3%）
- **加仓信号**：突破3,450（假突破）
- **加仓仓位**：5,000 USDT（50%）
- **加仓价格**：3,450
- **整体成本**：3,300
- **整体止损**：3,100
- **结果**：假突破下跌，止损亏损6.1%
